Executive summary

The malware demonstrates persistent execution through registry manipulation by creating a Run key entry via the _createRegistryRunEntry function. This ensures the malware survives system reboots. The technique involves invoking the Windows API functions RegOpenKeyExA, RegSetValueExA, and related registry APIs to write a malicious executable path into the 'Software\Microsoft\Windows\CurrentVersion\Run' registry hive. This behavior occurs during initial infection and is classified as malicious with 95% confidence. The purpose is to maintain long-term presence on the compromised host. The malware establishes command-and-control (C2) communication using _openShellConnection which leverages Winsock APIs such as WSASocketA, WSAStartup, and WSAConnect to connect to the remote IP address 188.127.247.130. It executes a ping command via cmd.exe to verify network connectivity before proceeding. This activity indicates an attempt to establish covert communication channels for data exfiltration or remote control. The implementation uses standard Windows networking libraries including WS2_32.dll and relies on low-level socket operations to achieve its objective. Upon deployment, the malware creates a new folder structure and copies itself to a hidden location using _createFolderAndCopySelf. This process utilizes CopyFileA and CreateDirectoryA APIs to place a copy of the binary under '%s\personal\windowsupdate.exe'. The intent is to avoid detection by placing the payload in a non-standard directory and renaming it to mimic legitimate software. This behavior is part of the initial infection and persistence strategy, occurring early in the execution chain. Finally, the malware exhibits anti-analysis behaviors through the use of obfuscated strings and dynamic library loading techniques. Functions like _loadLibrary and _GetProcAddress are used to dynamically resolve APIs, while string encoding and pseudo-relocation handling suggest attempts to evade static analysis. These mechanisms are implemented across various modules including ADVAPI32.dll, KERNEL32.dll, and WS2_32.dll, indicating a sophisticated approach to bypass sandboxing and heuristic detection systems.
